PROJECT DOCUMENTATION - 4WD MECANUM ROBOT (ESP32 + RASPBERRY PI 5)

1) PROJECT OVERVIEW

This robot is a 4-wheel mecanum mobile base (X-configuration) using DC motors with
encoders for each wheel, controlled by an ESP32 Dev Board (38-pin type), with a
Raspberry Pi 5 (8GB) as the high-level computer.

Main drive power supply (user-provided): 12V DC.

The robot also includes a NEMA17 stepper motor that drives a power screw, with a
horizontal limit switch module for homing.

Control goals include:
- Omni-directional motion and rotation
- Closed-loop wheel speed control
- Position and orientation control using PID in (x, y, theta)
- IMU-based orientation awareness (MPU6050 / IMU6050)

Robot frame convention (user-confirmed):
- +X axis points forward.


2) USER-PROVIDED MECHANICAL PARAMETERS

- Wheel radius: 97 mm = 0.097 m
- Left-right wheel spacing: 294 mm = 0.294 m
- Front-back wheel spacing: 180 mm = 0.180 m

For mecanum equations (common notation):
- Half lateral distance W = 0.294 / 2 = 0.147 m
- Half longitudinal distance L = 0.180 / 2 = 0.090 m
- Kinematic term (L + W) = 0.237 m

Note:
The naming wheelbase/track-width often changes between sources. The values above are
kept exactly as provided and converted for implementation use.


3) COMPUTE ARCHITECTURE

3.1 ESP32 (low-level real-time)
- Reads all 4 quadrature encoders
- Drives all 4 DC motors
- Runs wheel-level PID loops
- Executes mecanum inverse kinematics
- Handles stepper low-level control if assigned
- Enforces safety timeout and emergency stop behavior

3.2 Raspberry Pi 5 (high-level)
- High-level planning, user interface, and autonomy logic
- Sends motion commands to ESP32 through UART
- Logs telemetry and diagnostics

3.3 Communication
- ESP32 <-> Raspberry Pi: UART (as specified by user)

Use in control:

- yaw feedback into robot orientation PID


9) CONTROL REQUIREMENTS

User requirement:
- PID control for robot position and orientation: (x, y, theta)

Recommended control hierarchy:
- Outer loop: pose PID on x, y, theta -> outputs body velocity commands (vx, vy, wz)
- Inner loop: wheel speed PID for each motor -> tracks wheel targets from mecanum IK

Kinematic mapping overview:
- Input: desired (vx, vy, wz)
- Output: target wheel angular velocities (w_fl, w_fr, w_bl, w_br)
- Uses wheel radius and (L + W)


10) SPEED TARGET POLICY

User request indicates full-speed operation preference.

Engineering policy for safe implementation:
- Command saturation at maximum allowed motor duty
- Apply acceleration/ramp limiting to avoid wheel slip and large current spikes
- Keep PID outputs smooth while still allowing high peak speed

Electrical context:
- With a 12V supply and 12V-rated motors, full speed command should map to the
  validated max duty that does not brownout ESP32 or overheat drivers.


11) SOFTWARE DEVELOPMENT MODE

13) HOW TO EXTRACT THE UNKNOWNS QUICKLY

13.1 Encoder counts per wheel revolution
- Mark one wheel with tape
- Rotate exactly one wheel revolution by hand
- Read encoder count delta
- Repeat 5 times and average

13.2 Effective gear ratio (practical method)
- If encoder is on motor shaft, rotate wheel one full turn and measure motor-side counts
- Compare to encoder CPR assumptions to estimate reduction

13.3 DRV8825 microstepping
- Record M0/M1/M2 wiring (HIGH/LOW)
- Map wiring state using the table in Section 6


14) FREE RTOS TARGET TASK SPLIT (FINAL ARCHITECTURE)
